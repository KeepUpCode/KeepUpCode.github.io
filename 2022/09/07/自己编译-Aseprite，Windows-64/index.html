自己编译 Aseprite，Windows 64

KeepUP
2022-09-07

下载 Aseprite 源码

去到 Aseprite Github 仓库，下载源码 Aseprite-v-Source.zip

下载 CMake

到 CMAke 官网 根据自己的系统下载最新的版本

下载 ninja

去到 ninja Github 仓库，根据自己系统下载 ninja

下载 skia

去到 skia Github 仓库，根据自己系统下载 skia

安装 CMake

安装过程中注意选择加到环境变量里，安装目录默认即可

安装 Visual Studio

下载 Visual Studio Community 2022 ，并安装，安装时注意添加 Desktop development with C++ item 和 Windows 10.0.18362.0 SDK

编译前准备

1、解压 ninja 文件，里边只有一个 exe，为了方便我直接放到了 Cmake\bin 目录下，即默认安装目录 C:\Program Files\CMake\bin\
2、解压 skia ，并放到一个目录下，后边要用到，我放到了 D:\Skia\ 目录下
3、解压 Aseprite 源码，放到某一目录，例如 D:\Aseprite\

编译

打开 cmd 执行，中间是你安装 Visual Studio 的目录

然后到你的 Aseprite 源码目录下执行

DSKIA_DIR、DSKIA_LIBRARY_DIR、DSKIA_LIBRARY 中替换为你存放 skia 的目录
命令执行中间可能会有很多 failed / not found / warning 等，这些可以无视。只要不是 error 就可以
最后我们会在 build\bin\ 目录下生成 aseprite.exe，这就说明我们编译成功了
后续更新版本的话，我们删除 build\ 目录下文件，然后重新执行编译命令就可以了
